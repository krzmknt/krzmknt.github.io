構成

  • SSG: Hugo
    • 「Static Site Generator」で Google 検索したら２つ目に次のサイトが出てきた
      • 静的サイトジェネレーター Gatsby - Qiita
      • はじめ Gatsby にしてみたけどよくわからなかったので10分ぐらい触った後なんとなく３つ目の Hugo を使うことにした
      • hugo new site naomo hugo new posts 20210220.md hugo の3コマンドで ./public に公開するファイルが作成される
        • あとはこれをデプロイするだけなのでシンプル
    • Theme: hugo-tanaka
      • シンプルでスッキリしている
  • ホスティング: GitHub Pages
    • public 下を github.com/username/username.github.io に配置するだけで公開
  • ドメイン取得: お名前.com
  • DNS: Cloud Flare

CI

  • CircleCi
    • CIサービス初めて使った
    • CIとは、コミットをトリガーとしてビルド、テストを自動で行うことを指す模様
    • 今回はコミットをトリガーとして Hugo のビルドと Github Pagesへのデプロイを自動で行うようにした
      • 「Hugo CircleCi」 とか検索すればブログ記事がいくつも見つかる
    • 参考 Hugo, Github Pages, CircleCIつかってブログ構築 • abekoh’s tech note
    • .circleci/config.yml
version: 2.1
orbs:
  hugo: circleci/hugo@0.4.1
  slack: circleci/slack@4.3.0
jobs:
  deploy:
    docker:
      - image: cibuilds/base
    steps:
      - add_ssh_keys:
          fingerprints:
            - "b8:23:b7:0c:4c:3a:fc:b1:a6:c2:4e:8c:57:15:b9:d0"
      - attach_workspace:
          at: .
      - deploy:
          name: deploy to Github Pages
          command: |
            [ -e ~/.ssh ] || mkdir ~/.ssh
            echo -e "Host github.com\n\tStrictHostKeyChecking no\n" > ~/.ssh/config

            DEPLOY_DIR=deploy
            git config --global user.email $USER_EMAIL
            git config --global user.name $CIRCLE_USERNAME
            git clone git@github.com:krzmknt/krzmknt.github.io.git $DEPLOY_DIR

            cd $DEPLOY_DIR
            rm -vrf ./*
            cp -v -R ../public/* ./

            echo "naomo.jp" > CNAME

            git add -f .
            git commit -m "Deploy #$CIRCLE_BUILD_NUM from CircleCI [ci skip]"
            echo "Lets push"
            git push origin master -f            
      - slack/notify:
          event: fail
          template: basic_fail_1
      - slack/notify:
          event: pass
          template: success_tagged_deploy_1
workflows:
  version: 2.1
  main:
    jobs:
      - hugo/build:
          html-proofer: false
      - deploy:
          requires:
            - hugo/build
          filters:
            branches:
              only: master
          context: Slack Notification
